# 书本索引系统核心概念说明

本系统通过建立一套标准化的 ID 体系，旨在解决古籍及现代文献在数字化过程中面临的层级归属复杂、版本关联困难等问题。系统核心由以下三个概念组成：

## 一、 核心概念定义

### 1. 作品 (Work)
* **定义**：指代文献的**抽象知识内容**。它不随装订形式、印刷质量或出版年代的变化而改变。
* **作用**：作为“灵魂”层，用于聚合所有相同内容的不同版本。
* **示例**：
    * 《红楼梦》
    * 《史记》
    * 《论语》

### 2. 丛编 (Collection)
* **定义**：指代文献的**组织容器或集合关系**。
* **特性**：具有**递归性**。一个大型丛编可以包含多个子丛编，最终指向具体的“书”。
* **示例**：
    * **大丛编**：`《四库全书》文渊阁本`
    * **子丛编**：`《史记》文渊阁本`（属于四库全书中的一个组成部分）

### 3. 书 (Book)
* **定义**：指代现实中**具体装订、印刷的最小物理实体**。
* **作用**：是系统中被索引的最基本单元，直接对应到具体的扫描件、版本特征或实物馆藏。
* **示例**：
    * `《红楼梦》甲戌本`（一整套物理书）
    * `《红楼梦》程甲本 第一册`

---

## 二、 概念间的逻辑关系

系统中各层级通过 ID 进行关联，形成以下逻辑链条：

1. **作品 ↔ 丛编/书**：一个“作品”可以对应多个“丛编”中的不同版本，也可以对应多本独立的“书”。
2. **丛编 ↔ 丛编**：丛编支持多级嵌套（大套含小套）。
3. **丛编 ↔ 书**：丛编是书的容器，负责将具体的“书”按逻辑组织起来。
